# Header Footer Nav Cleanup Plan

## 1. 目标与最终产物

本次是一个独立的新任务，只处理页面顶部导航和底部页脚收口，不改动上一份“首页入口 + 模型级详情页”的主任务边界。

最终产物：

- 页面右上角新增两个快捷链接：
  - `首页`
  - `检测方法`
- 删除顶部现有的 `联系我们`
- 删除底部整块 `平台声明`
- 保留必要底部链接，不再保留说明卡片区

## 2. 本次只处理的范围

只改这几个点：

- `frontend/src/components/Header.tsx`
- `frontend/src/components/Footer.tsx`
- 必要时补充少量文案或路由链接逻辑

本次不处理：

- 首页表格结构
- 模型级详情页表格结构
- `rpdiag` 数据接入
- `/contact` 页面本身是否保留

## 3. 需求拆解

### 3.1 顶部导航

页面右上角增加两个快捷链接：

- `首页`
- `检测方法`

要求：

- 使用现有顶部 Header 区域
- 不新增第二套顶栏容器
- 视觉参考用户给出的 RPDiag 截图：
  - 深色背景
  - 文字型导航
  - 当前页高亮

链接语义：

- `首页` -> `/`
- `检测方法` -> `http://localhost:8080/p/claudecn-gpt?service=cc&channel=78%3AClaudeCN-gpt`

### 3.2 删除联系我们

删除 Header 中现有的：

- 桌面端 `联系我们`
- 移动端 `联系`

要求：

- 不保留占位按钮
- 不替换为其他营销或联系入口

### 3.3 删除平台声明

删除 Footer 中整块：

- `平台声明` 标题
- 4 张说明卡片
- 移动端折叠逻辑

Footer 只保留必要内容：

- `GitHub`
- `问题反馈`
- 版本信息

如果保留“质量/检测”入口，必须与顶部导航语义一致，不能形成第二套主导航。

## 4. 实施顺序

1. 审计 Header 当前桌面端和移动端入口结构
2. 新增 `首页 / 检测方法` 顶部快捷链接
3. 删除 `联系我们` 桌面和移动端入口
4. 精简 Footer，删除平台声明和折叠逻辑
5. 本地 build
6. `localhost:8080` 运行态验收

## 5. 验收标准

必须同时满足：

1. 页面右上角出现 `首页 / 检测方法`
2. `联系我们` 不再出现
3. `平台声明` 不再出现
4. 顶部和底部不存在重复冲突的主导航
5. 前端构建通过
6. `localhost:8080` 可直接看到结果
